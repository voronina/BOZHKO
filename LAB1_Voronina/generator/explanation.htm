Генератор заданий

Сценарий использования

Исходные данные

Для получения задания введите свой вариант. После этого в текстовом поле "Исходные данные" появится json-код, описывающий операционное пространство в следующем формате: 

{
	"start" = Point,
	"finish" = Point,
	"polygons" = [
		{
			"x",
			"y",
			"vertices" = [
				Point,
				...
			]
		},
		...
	]
}
где Point - объект, состоящий из двух полей "x" и "y", хранящих координаты описываемой точки. Точка "start" и точка "finish" совпадают для всех вариантов и имеют координаты {2, 2} и {48, 28} соответственно. Массив "polygons" хранит объекты, описывающие все сгенерированные препятствия. Каждое препятствие представлено в виде набора точек, записанных в порядке радиального обхода. Помимо точек, задающих полигон отдельно записаны координаты точки, которая заведомо находится внутри полигона. Такая точка может быть использована для упрощения алгоритма проверки точки на принадлежность полигону.

Отображение результатов

Для отображения результатов предусмотрено два типа примитивов: прямяе и точки. Все они передаются в виде массива в текстовое поле "Решение". Это поле можно использовать как для вывода пути от начала к концу, как показано в примере, появляющемся при открытии страницы, так и для отображения более сложных иллюстраций, например, графов.

Разъяснения относительно дискретности

Несмотря на то, что все исходные данные выдаются в целых числах, а также на фоне отображается сетка, генератор допускает обработку решений, представленного в виде точек и прямых, пимеющих не целые координаты. Таком образом для алгоритмов, исследующих операционное пространство как непрерывное, результат могут быть отображены корректно. Для алгоритмов, ориентированных на дискретные пространства можно игнорировать этот факт, считая исследуемое пространство дискретным и представляя результаты в виде целых чисел, как это показано на иллюстрации
